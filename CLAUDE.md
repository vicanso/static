This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

`static-serve` is a high-performance static file server written in Rust (edition 2024, Axum) aimed at serving frontend static assets. Storage is abstracted through `opendal` (local filesystem, S3, FTP, MongoDB GridFS). Features: TinyUFO LRU cache, gzip/brotli/zstd compression, pre-compressed file support, ETag/304, HTTP range requests + If-Range, directory auto-indexing, HTML content replacement, CORS, Basic Auth, IP allow/block lists, per-IP rate limiting, redirects, nosniff default, Prometheus `/metrics`, text/JSON access logs, configurable graceful SIGTERM drain.

## Commands

```bash
make dev      # cargo watch, STATIC_PATH=./assets, RUST_LOG=INFO  (needs cargo-watch)
make lint     # typos + cargo clippy --all-targets --all -- --deny=warnings
make fmt      # cargo fmt
make release  # cargo build --release  (LTO/strip/panic=abort come from Cargo.toml [profile.release])
make bloat    # cargo bloat --release
```

Unit tests live in-module under `#[cfg(test)] mod tests` in `serve.rs`, `metrics.rs`, and `rate_limit.rs` (pure logic: range parsing, `Accept-Encoding` negotiation, `If-Range`, cache-key formatting, latency bucketing, the token-bucket math). Run all with `cargo test`, or one with `cargo test <name>`. There is no `tests/` integration dir yet. `unwrap_used = "deny"` applies to test code too, so tests use `assert!`/`assert_eq!`/`matches!` and infallible constructors rather than `.unwrap()`; the rate limiter exposes a private time-injectable `check_at(ip, now)` so its refill math is tested deterministically without a real clock.

`make lint` is the gate that matters and must pass. Beyond `--deny=warnings`, `Cargo.toml` sets `[lints.clippy] unwrap_used = "deny"` — **do not use `.unwrap()`**; propagate via the `Error` type / `Result`. Run `cargo clippy` (or `make lint`) before considering any change done.

## Architecture

Seven Rust modules (`src/*.rs`) plus compile-time HTML templates. Request flow: `main.rs` fallback route → `serve.rs static_serve()`/`get_file()` → cache check → path validation → opendal `stat` → directory/file handling → header generation → ETag/304 → body (buffered or streamed).

- **`config.rs`** — All `STATIC_*` env vars via `envy`, parsed once in `Config::new()`. **On any parse error it logs and `std::process::exit(1)` — deliberately no silent fallback to defaults.** Duration/byte-size fields (`timeout`, `cache_ttl`, `read_max_size`, `shutdown_delay`) are `Option<String>` parsed by hand (`parse_duration_or_exit`/`parse_bytesize_or_exit`); `STATIC_CORS_MAX_AGE` is likewise validated (digits only) with exit-on-bad-value. **Do not convert these back to `#[serde(deserialize_with)]`**: combined with the struct's `#[serde(default)]`, `envy 0.4` then fails whenever those vars are absent, which previously made the entire config silently revert to defaults. Prefix-scanned vars (`STATIC_HTML_REPLACE_*`, `STATIC_RESPONSE_HEADER_*`, `STATIC_CACHE_CONTROL_EXT_*`, `STATIC_REDIRECT_*`, `STATIC_BASIC_AUTH_*`) are handled by a manual `std::env::vars()` loop, not the `EnvConfig` struct.
- **`storage.rs`** — `opendal` abstraction; one `Storage` in a `OnceLock`. Backend auto-detected from `STATIC_PATH` (read here, separately from `config.rs`): bare path → local FS; `http(s)://...?bucket=` → S3; `ftp://` → FTP; `mongodb://` → GridFS. Local FS path is canonicalized; `validate()` blocks path traversal (resolved path must stay under root). `validate()` does a lexical `..` check (always on) plus a `canonicalize()` syscall to catch symlinks escaping the root — the latter is skippable via `STATIC_DISABLE_SYMLINK_CHECK=true` (read once into a `OnceLock`, exit-on-bad-value) for symlink-free asset trees that don't want the per-miss syscall.
- **`serve.rs`** — Core handler. TinyUFO LRU cache; **HTML and directory autoindex listings are never cached** (mutable across deploys/requests via opendal backends). Buffers files `< STATIC_READ_MAX_SIZE` (default 250KB), streams larger ones; supports range/206 (a single range streams or slices one body; multiple comma-separated ranges return a `multipart/byteranges` body assembled in memory via `build_multipart_byteranges`, capped at `MAX_RANGES` = 100 with excess → full 200; multi-range on a Content-Encoded representation falls back to a full 200 since the envelope can't itself be encoded), `If-Range`, 304, per-extension cache-control, and `.br`/`.zst`/`.gz` pre-compressed negotiation (q-value aware via `EncodingPrefs` — `br;q=0` is honored as a refusal). **The in-memory cache key is encoding-aware** (`encoding_cache_key` = `"<path>\0<encoding>"`; identity = empty suffix), so pre-compressed responses ARE cached — each negotiated encoding is a distinct entry, and a hit returns the exact bytes + `Content-Encoding` without re-`stat`ing the path or probing its `.br`/`.zst`/`.gz` siblings. A lookup probes the client's accepted encodings (br>zstd>gzip) then identity; first hit wins. **If you add a new way the served bytes vary by request, fold it into the cache key** or you'll serve the wrong variant. Metadata-only entries (large/streamed files, `body == None`) are cached too so a hit skips the `stat`+sibling probes; HEAD responses are never cached (their body is always None). **Cache misses are single-flighted** (`get_file` → `Flight`/`INFLIGHT`): concurrent misses for the same `singleflight_key` (path + accepted-encoding bits + HEAD-ness) collapse to one leader load; followers share the leader's `Arc<FileInfo>` (streamed bodies still open their own reader per follower). Only `Ok` results are shared — errors/redirects fall through to an independent load, so request-specific data (e.g. a 301 `Location` built from the request path) is never cross-fed. File responses always carry `Vary: Accept-Encoding` (the global CompressionLayer may compress on the fly regardless of `STATIC_PRECOMPRESSED`). `If-Range` uses strong validators only — a weak ETag never satisfies it (RFC 7233), so the Range is ignored and a full 200 is returned. Autoindex file names are HTML-escaped and percent-encoded (`html_escape`) — they are attacker-influenced. The `.wasm`/`.mjs` MIME override only fills in when the type is otherwise undetermined (never clobbers a backend-set Content-Type). Cache hits/misses are counted via `crate::metrics`.
- **`error.rs`** — `Error` enum (`Unknown`, `InvalidFile`, `Timeout`, `NotFound`, `Forbidden`, `Openedal`, `ParseUrl`). `IntoResponse` renders a **built-in HTML error page for every error status**; internal/opendal detail (paths, OS errors) is **never** sent to the client, only logged server-side. Override via `STATIC_ERROR_PAGE` (filesystem path; template uses `{{STATUS}}`/`{{REASON}}`), resolved once at startup by `init_error_template()` (called from `main()`); set-but-unreadable → process exits. Independent from a user `404.html` in `STATIC_PATH` (404-only, served verbatim, takes precedence for 404s).
- **`metrics.rs`** — Process-memory `AtomicU64` metrics rendered as Prometheus text: monotonic counters (requests, responses by status class, response bytes, cache hits/misses), a `static_serve_request_duration_seconds` **histogram** (per-bucket atomics summed cumulatively at render; observed in `track_metrics` as time-to-produce-the-response, **not** including streamed-body transfer), and **gauges** `static_serve_cache_entries` (live occupancy) + `static_serve_cache_capacity` (configured max). The occupancy gauge is maintained in `serve.rs`'s `set_file_to_cache` (TinyUFO has no `len()`): a pre-`put` `get` decides if the key is new (+1), and `put`'s returned evicted-entries `Vec` gives the count to subtract — correct across both fresh inserts and re-caches of logically-expired-but-still-present keys. Capacity is set once at startup via `set_cache_capacity`. Exposed at `GET /metrics` when `STATIC_METRICS=true` (default). **`STATIC_METRICS` is a master switch** (`set_enabled` → the `ENABLED` `OnceLock<bool>` — write-once at startup, no runtime toggle): when false, every `record_*` returns immediately *and* callers skip their own metrics-only setup (`track_metrics` skips the two `Instant::now()` clock reads; `set_file_to_cache` skips the gauge's pre-`put` `get`) — so disabling metrics removes both the route and the per-request collection cost, not just the route. No external deps; `Relaxed` ordering (scrape metrics need no cross-counter consistency).
- **`rate_limit.rs`** — Per-IP token-bucket limiter (`STATIC_RATE_LIMIT` req/sec, `STATIC_RATE_LIMIT_BURST` capacity; `0` = disabled, the default). `STATIC_RATE_LIMIT_EXEMPT` (comma-separated IPs/CIDRs, parsed by `config.rs`'s `parse_ip_list`) bypasses the limiter — the exemption is enforced in `main.rs` `serve` (where the `Config` IpNet list and resolved `ClientIp` live), not in this module. A single global `OnceLock<Option<RateLimiter>>`; `check(ip)` returns `Some(retry_after_secs)` to reject with 429 or `None` to allow — **lock-free and zero-cost when disabled** (returns before touching the mutex). One `Mutex<HashMap<IpAddr, Bucket>>` holds the buckets (lazy float refill per request); idle buckets that have refilled to full capacity are swept every 60s so the map stays bounded to active IPs. No new deps — deliberately not `tower_governor`, so limiting reuses the caller's XFF-aware `ClientIp` rather than a second IP-extraction path. The burst clamps to `>= 1` and the limiter is only built when rate `> 0`, so the refill division never hits a zero rate.
- **`main.rs`** — Axum router: `GET /health`, optional `GET /metrics`, catch-all `fallback(serve)` (a bare handler, **not** `get(serve)`, so `OPTIONS` reaches the handler for CORS preflight; other non-GET/HEAD methods get 405). Middleware order matters (outermost→in): `track_metrics` (records a sample when metrics are enabled, and always strips the internal `x-original-size` header so it never leaks to clients) → optional `access_log` → `HandleErrorLayer` → optional `CompressionLayer` (only when `compress_min_length > 0`; `0` disables compression entirely) → timeout → handler. In `serve`: IP block/allow → **per-IP rate limit (bare 429 + `Retry-After`, after IP control so blocked IPs allocate no bucket state; IPs/CIDRs in `STATIC_RATE_LIMIT_EXEMPT` skip it; no-op when disabled)** → **CORS preflight / method gate (before Basic Auth, so a credential-less browser preflight is not 401'd)** → Basic Auth → redirects → file serving. `apply_common_headers` adds `STATIC_RESPONSE_HEADER_*`, the `X-Content-Type-Options: nosniff` default (skipped if you set it yourself), and CORS headers to every served/redirect/404 response. Graceful SIGTERM: `/health` returns 503 (Service Unavailable, not 500 — LBs treat it as a temporary drain, not an alert-worthy error) for a `STATIC_SHUTDOWN_DELAY` drain window (default 5s) before shutdown. `LOG_FORMAT=json` switches the tracing subscriber to JSON.

**Templates:** `src/templates/error.html` and `src/templates/autoindex.html` are embedded at compile time via `include_str!` (paths relative to `src/error.rs` / `src/serve.rs`). Runtime placeholders: `{{STATUS}}`/`{{REASON}}` (error), `{{CONTENT}}` (autoindex). Edit the `.html` files directly — no Rust change needed.

## Configuration & Non-Obvious Behavior

Full env var reference lives in `README.md` / `README_zh.md`. Key behaviors not obvious from the code:

- HTML always gets `Cache-Control: no-cache` regardless of `STATIC_CACHE_CONTROL`, and is never cached in memory.
- Cache-Control precedence: HTML `no-cache` > backend-provided > `STATIC_CACHE_CONTROL_EXT_<ext>` > `STATIC_CACHE_CONTROL`.
- `STATIC_COMPRESS_MIN_LENGTH=0` removes the compression layer entirely (it does not mean "compress everything").
- `/health` and `/metrics` are separate routes — they bypass Basic Auth, IP access control, and rate limiting. `STATIC_METRICS=false` removes the `/metrics` route **and** disables per-request metric collection (the `track_metrics` layer still runs to strip the internal `x-original-size` header, but records nothing).
- CORS preflight (`OPTIONS`) is answered before Basic Auth so a credential-less browser preflight is never 401'd; it still passes IP access control.
- `X-Content-Type-Options: nosniff` is added by default (`STATIC_CONTENT_TYPE_NOSNIFF=true`) but never overrides a value you set via `STATIC_RESPONSE_HEADER_*`.
- File responses always carry `Vary: Accept-Encoding`; CORS allowlist matches append `Vary: Origin`. Autoindex listings are `no-cache` and never stored in the in-memory cache.
- The 408 timeout page is generated outside the compression layer (uncompressed); other error pages are inside it. Error responses do not pass through `apply_common_headers`, so they carry no CORS/nosniff headers.

## Conventions

- Conventional Commits (`feat:`, `fix:`, `chore:`); history commits directly to `main`.
- `README.md` and `README_zh.md` are a mirrored pair — keep structure and content in sync when changing config or behavior docs.
- **Bring cross-module items into scope with `use`; do not call them through an inline fully-qualified `crate::...` path.** Write `use crate::metrics;` once at the top and call `metrics::record_cache_insert(...)`, not `crate::metrics::record_cache_insert(...)` at the call site. (`use crate::module;` and calling `module::item()` is preferred over importing bare functions, so the originating module stays visible at the call site — this matches `main.rs`.) `crate::` belongs only in `use` statements.
